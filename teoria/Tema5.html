Data frames

Data frame. Un data frame es una tabla de doble entrada, formada por variables en las columnas y observaciones de estas variables en las filas, de manera que cada fila contiene los valores de las variables para un mismo caso o un mismo individuo.

  • data(): para abrir una ventana con la lista de los objetos de datos a los que tenemos acceso en la sesi√≥n actual de R (los que lleva la instalaci√≥n b√°sica de R y los que aportan los paquetes que tengamos cargados.

    • Si entramos data(package=.packages(all.available = TRUE)) obtendremos la lista de todos los objetos de datos a los que tenemos acceso, incluyendo los de los paquetes que tengamos instalados, pero que no est√©n cargados en la sesi√≥n actual.

Obteniendo informaci√≥n del data frame

  • tail(d.f,n): para mostrar las \(n\) √∫ltimas filas del data frame. Por defecto semuestran las 6 √∫ltimas
  • str(d.f): para conocer la estructura global de un data frame

Obteniendo informaci√≥n del data frame

  • rownames(d.f): para producir un vector con los identificadores de las filas
    • R entiende siempre que estos identificadores son palabras, aunque sean n√∫meros, de ah√≠ que los imprima entre comillas
  • colnames(d.f): para producir un vector con los identificadores de las columnas
  • dimnames(d.f): para producir una list formada por dos vectores (el de los identificadores de las filas y el de los nombres de las columnas)
  • nrow(d.f): para consultar el n√∫mero de filas de un data frame
  • ncol(d.f): para consultar el n√∫mero de columnas de un data frame
  • dim(d.f): para producir un vector con el n√∫mero de filas y el de columnas

Obteniendo informaci√≥n del data frame

  • d.f$nombre_variable: para obtener una columna concreta de un dataframe
    • El resultado ser√° un vector o un factor, seg√∫n c√≥mo est√© definida la columna dentro del data frame
    • Las variables de un data frame son internas, no est√°n definidas en el entorno global de trabajo de R

Leyendo tablas de datos

  • read.table(): para definir un data frame a partir de una tabla de datos contenida en un fichero
    • Este fichero puede estar guardado en nuestro ordenador o bien podemos conocer su url. Sea cual sea el caso, se aplica la funci√≥n al nombre del fichero o a la direcci√≥n entre comillas

Aqu√≠ ten√©is una lista de data frames para practicar

Par√°metros de read.table()

  • header = TRUE: para indicar si la tabla que importamos tiene una primera fila con los nombres de las columnas. El valor por defecto es FALSE
  • col.names = c(...): para especificar el nombre de las columnas. No olvid√©is que cada nombre debe ir entre comillas
  • sep: para especificar las separaciones entre columnas en el fichero (si no es un espacio en blanco). Si es as√≠, hay que introducir el par√°metro pertinente entre comillas
  • dec: para especificar el signo que separa la parte entera de la decimal (si no es un punto. Si es as√≠, hay que introducir el par√°metro pertinente entre comillas

M√°s par√°metros de read.table()

  • stringsAsFactors: para prohibir la transformaci√≥n de las columnas de palabras en factores debemos usar stringsAsFactors=FALSE (ya que por defecto, R realiza dicha transformaci√≥n)

Leyendo diferentes tipos de fichero

  • read.csv(): para importar ficheros en formato CSV
  • read.xls() o read.xlsx(): para importar hojas de c√°lculo tipo Excel u OpenOffice en formato XLS o XLSX, respectivamente. Se necesita el paquete xlsx
  • read.mtb(): para importar tablas de datos Minitab. Se necesita el paquete foreign
  • read.spss(): para importar tablas de datos SPSS. Se necesita el paquete foreign

Construyendo data frames

  • data.frame(vector_1,...,vector_n): para construir un data frame a partir de vectores introducidos en el orden en el que queremos disponer las columnas de la tabla
    • R considera del mismo tipo de datos todas las entradas de una columna de un data frame
    • Las variables tomar√°n los nombres de los vectores. Estos nombres se pueden especificar en el argumento de data.frame entrando una construcci√≥n de la forma nombre_variable = vector
    • rownames: para especificar los identificadores de las filas
    • Tambi√©n en esta funci√≥n podemos hacer uso del par√°metro stringsAsFactors para evitar la transformaci√≥n de las columnas de tipo palabra en factores

Construyendo data frames

  • d.f$new_var: para a√±adir una nueva variable al data frame
    • Podemos concatenar columnas con un data frame existente mediante la funci√≥n cbind(). De este modo se puede a√±adir la columna directamente sin necesidad de convertirla antes a data frame
    • Esta nueva variable ha de tener la misma longitud que el resto de columnas del data frame original. Si no, se a√±adir√°n valores NA a las variables del data frame original o a la nueva variable hasta completar la misma longitud

Cambiando los tipos de datos

  • as.character: para transformar todos los datos de un objeto en palabras
  • as.integer: para transformar todos los datos de un objeto a n√∫meros enteros
  • as.numeric: para transformar todos los datos de un objeto a n√∫meros reales

M√°s sobre sub-data frames

  • droplevels(d.f): para borrar los niveles sobrantes de todos los factores, ya que las columnas que son factores heredan en los sub-data frames todos los niveles del factor original, aunque no aparezcan en el trozo que hemos extra√≠do

  • select(d.f, par√°metros): para especificar que queremos extraer de un data frame

    • starts_with("x"): extrae del data frame las variables cuyo nombre empieza con la palabra “x”
    • ends_with("x"): extrae del data frame las variables cuyo nombre termina con la palabra “x”
    • contains("x"): extrae del data frame las variables cuyo nombre contiene la palabra “x”
    • Se necesita el paquete dplyr o mejor a√∫n tidyverse

M√°s sobre sub-data frames

  • subset(d.f,condici√≥n,select = columnas): para extraer del data frame las filas que cumplen la condici√≥n y las columnas especificadas
    • Si queremos todas las filas, no hay que especificar ninguna condici√≥n
    • Si queremos todas las columnas, no hace especificar el par√°metro select
    • Las variables en la condici√≥n se especifican con su nombre, sin a√±adir antes el nombre del data frame

Aplicando funciones a data frames

  • sapply(d.f, funci√≥n): para aplicar una funci√≥n a todas las columnas de un data frame en un solo paso
    • na.rm=TRUE: para evitar que el valor que devuelva la funci√≥n para las columnas que contengan alg√∫n NA sea NA
  • aggregate(variables~factors,data=d.f,FUN=funci√≥n): para aplicar una funci√≥n a variables de un data frame clasificadas por los niveles de un, o m√°s de un, factor
    • Si queremos aplicar la funci√≥n a m√°s de una variable, tenemos que agruparlas con un cbind
    • Si queremos separar las variables mediante m√°s de un factor, tenemos que agruparlos con signos \(+\)

Variables globales

  • attach(d.f): para hacer que R entienda sus variables como globales y que las podamos usar por su nombre, sin necesidad de a√±adir delante el nombre del data frame y el s√≠mbolo $
    • Si ya hubiera existido una variable definida con el mismo nombre que una variable del data frame al que aplicamos attach, hubi√©ramos obtenido un mensaje de error al ejecutar esta funci√≥n y no se hubiera reescrito la variable global original
  • detach(d.f): para devolver la situaci√≥n original, eliminando del entorno global las variables del data frame
